# StudyGold MVP LP UI Tone & Palette

## 1. デザインコンセプト

**頑張りを、安心して使えるご褒美に変える。**

正式ブランド名は **StudyGold** とする。

StudyGoldのLPは、学習や努力の記録をただの数値ではなく「自分を労るための資産」として見せる。黒基調の集中感に、ゴールドの報酬感、実写真の生活感を組み合わせて、ゲームっぽさと現実のご褒美感を両立する。

MVPでは機能を説明しすぎず、以下の体験が一目で伝わることを優先する。

- 学習時間や成果がゴールドに変わる
- 自分で設定したご褒美と交換できる
- 残高が増減し、達成感が可視化される
- 獲得・交換時に気持ちが上がる演出がある

## 1-1. 架空ブランド名案

| Name | 読み | 状態 | 方向性 |
| --- | --- | --- | --- |
| StudyGold | スタディゴールド | 採用 | 学習成果がゴールドに変わることが最も直感的に伝わる。MVP向き。 |
| RestPay | レストペイ | 候補 | 頑張った分で休む、という価値が短く伝わる。少し大人っぽい。 |
| Rewardy | リワーディ | 候補 | ご褒美アプリらしい軽さがあり、親しみやすい。 |
| GoldBreak | ゴールドブレイク | 候補 | 学習後の休憩・ご褒美交換をゲーム的に見せやすい。 |
| YokuYatta | ヨクヤッタ | 候補 | 「よくやった」をブランド化。労いの感情が最も強い。 |

採用理由は、機能価値が一瞬で伝わり、添付UIのゴールド残高・報酬カードとの相性が良いため。

## 4. 色の使い分け

### LPファーストビュー

- 背景は `--gradient-bg`
- H1は `--color-text`
- 「ゴールド」「交換」「ご褒美」などの価値語だけ `--color-gold`
- CTAは `--gradient-gold`
- ヒーロー内にアプリ画面のモックを置く場合、カードは `--color-surface`

### アプリ画面モック

- 残高カードは大きく、`--color-surface-raised`
- GP数値は `--color-text`、単位や獲得差分は `--color-gold`
- 学習開始ボタンは `--color-gold`
- 交換ボタンは通常 `transparent + gold border`、交換可能な強調時のみ塗り

### ご褒美カード

- 写真を上部に大きく使う
- 写真下は `--color-surface`
- 価格は必ず `--color-gold`
- カテゴリバッジはカテゴリごとに色を分ける
- 残高不足カードは彩度を落とし、交換ボタンを弱める

## 6. UI Components Tone

### Buttons

- Primary: ゴールド塗り、太字、短い文言
- Secondary: ダーク背景 + ゴールド枠線
- Disabled: `--color-surface-soft` + `--color-text-dim`
- 角丸は8-14px程度。カードよりボタンのほうが少し丸い程度にする

### Cards

- 角丸は16-24px
- 背景は黒に近い面を使い、境界線は薄く
- 影は強すぎず、重要な報酬カードだけゴールドの光を足す
- カード内カードは避ける

### Badges

- カテゴリを一瞬で判断できる色にする
- テキストは短くする
- 例: `日常の贅沢`, `エンタメ`, `リフレッシュ`, `長期目標`

### Motion / Feedback

MVPのLPでは実装コストを抑え、以下の視覚演出に絞る。

- GP獲得: 数値が増えるアニメーション、金の粒が短く散る
- 交換: カードが一瞬光り、残高が減る表現
- CTA hover: ゴールドの光を少し強める
- Progress: ゴールドのバーが伸びる

音やハプティクスは本実装向けの価値としてLP内で表現し、MVP UIでは「演出の存在」が伝わる見せ方に留める。

## 7. 写真の選び方

ご褒美画像は「実際にこれと交換したい」と思える写真を使う。

- コーヒー、映画館、ゲーム環境、カフェ、外食、スニーカーなど実生活に近い写真
- 暗めの写真でも対象物が分かるもの
- 過度に抽象的な写真、背景だけの写真は避ける
- 高額ご褒美ほど写真の存在感を大きくする
- 自分で追加するご褒美は、将来的にスマホ写真アップロードを想定する

## 8. LP MVPの画面構成案

1. Hero
   - 「頑張りを、安心して使えるご褒美に。」
   - 学習時間がGPに変わるアプリ画面モック
   - CTA: `MVPを見てみる`

2. Problem
   - 学習後も休むことに罪悪感がある
   - 成果が見えづらく、継続がしんどい
   - ご褒美管理が別アプリに分かれて続かない

3. How It Works
   - 記録する
   - ゴールドが貯まる
   - ご褒美と交換する

4. Reward Shop Preview
   - 実写真のご褒美カードを並べる
   - 価格、カテゴリ、交換ボタンを見せる

5. Rich Feedback
   - 獲得時の光、音、ハプティクス予定を短く見せる

6. CTA
   - 「今日の頑張りを、ご褒美に変える」

## 9. コピーの方向性

### 良いコピー

- 今日の45分が、週末の映画に変わる。
- 頑張りを記録して、休む理由をつくる。
- 学習時間をゴールドに変えて、自分で決めたご褒美と交換。
- 休むことに、ちゃんと対価を。

### 避けるコピー

- 絶対に継続できる
- 怠けを防止する
- 自分を追い込もう
- 学習効率を最大化する

## 10. MVPで優先する見た目

- ゴールド残高が一番目に入ること
- ご褒美カードの写真が魅力的に見えること
- 「記録 -> 獲得 -> 交換」の流れが3秒で分かること
- 学習管理ではなく、自分を労るサービスに見えること
- LPでもアプリの利用イメージが具体的に伝わること
